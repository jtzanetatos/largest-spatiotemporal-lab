# INSTRUCTIONS: API Documentation

## Purpose

Document any public interface exposed by the project:

- REST/HTTP APIs (FastAPI, Flask, etc.)
- Python library APIs
- CLI entrypoints

## How to fill README.md

Use README.md as:

- A table of contents for API components
- A place to describe endpoints or modules
- A link to autogenerated docs (OpenAPI, sphinx, mkdocs)

## Best practices

- Keep API docs synchronized with actual code.
- For ML inference APIs, define request/response schemas using Pydantic.
- For Python APIs, document only the public surface.
